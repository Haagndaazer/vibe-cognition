FINDINGS (condensed):

1. [high/bug] ChromaDB double-recreate race: __init__ never stamps embed_scheme (only recreate_collection does), and each server's background thread checks a process-local startup-frozen metadata snapshot — so EVERY fresh install triggers the migration, and two same-project sessions in the 2-30s model-load window both call recreate_collection(); the second delete-recreates after the first synced, silently wiping its vectors (broad except swallows it). git_hygiene solves the identical one-time-migration problem with a file lock — inconsistent discipline. embeddings/storage.py:43-77, server.py:184-235.
2. [high/broken-assumption] Compact drops the entire prime digest: compact hook reinjects only static SERVER_INSTRUCTIONS; constraints/open-tasks/patterns/decisions/incidents are never restored and no cue says they're stale — defeats "the graph IS the backlog" in exactly the long sessions it serves. Reinject decision 7e9665028bb1 was scoped to instructions-survival only; data payload never weighed. hooks.json:5-24, instructions.py:50-63, prime.py:170-186.
3. [high/bug] Merge-shaped journal replay silently drops edges/removals: _replay_entry drops add_edge/remove_edge/remove_node with missing endpoints with NO log and the offset advances forever; merge=union (the supported separate-clones mechanism) can interleave lines edge-before-node. No merge-shaped replay tests exist. Maintainers use shared-checkout so this topology is never exercised. storage.py:1001-1029, 900-919.
4. [high/bug] Startup embedding reconciler is a drifted second implementation: _sync_cognition_embeddings reimplements the entity embed inline (omits task status/owner; routes workflows through the generic branch — no chunking) instead of calling _embed_entity_node/_embed_workflow ("THE single shared embed path", pattern c6f92230831c). Nodes created in the model-load window get permanently degraded vectors; workflows have no repair path (in-place edit refused by design). server.py:108-124 vs cognition_tools.py:58-136.
5. [medium-high/broken-assumption] REPO_PATH="" (present-but-empty) silently redirects the graph to the plugin install dir: pydantic-settings without env_ignore_empty treats empty as explicit, skipping the CLAUDE_PROJECT_DIR fallback; Path("") = cwd = plugin root (uv run --directory). Empirically verified. Three codebase reads are vulnerable (config/prime/backfill); only dashboard/cli.py's `or` guard is safe — never a decision. config.py:12-30, plugin.json:21.
6. [medium-high/gap] No episode dedup for shared commit refs: two clones independently mint distinct episodes for the same commit:<sha> (no find_nodes_by_ref before mint — documents DO sha-dedup); both survive merges forever; duplicate_of (the repair edge) is refused by tools AND forbidden by the curate skill; redirect_edges unreachable. cognition_tools.py:159-178, skills/vibe-curate/SKILL.md.
7. [medium/blindspot] Deterministic-edge backfill skips nodes with ANY edge, not nodes missing the deterministic edges their refs warrant — cross-clone linkage stays permanently partial; get_edgeless_nodes shares the coarse predicate. server.py:22-49.
8. [medium/gap] Chunk completeness probed by chunk-0 only — crash mid-chunking loop reads as fully synced forever; document permanently under-indexed with no signal. server.py:104-106.
9. [medium/blindspot] Settings env_file=".env" resolves against the shared plugin root (server cwd), not the project — per-project .env inert; plugin-root .env silently configures every project. config.py:23-30.
10. [medium/gap] No CI check that pyproject.toml and plugin.json versions match — hand-synced convention in a release flow already bitten by an adjacent drift class (a37a60664cfc).
11. [low-medium/broken-assumption] Document blob/sidecar written BEFORE the owning node is journaled (inverts C-4 journal-first) — crash orphans artifacts unreclaimably (delete-reclaim only walks down from a node; no orphan scanner); store_copy leaks into the git-committed tree. cognition_tools.py:537-698.
12. [low/gap] Onboarding stated in three independently-maintained channels (SERVER_INSTRUCTIONS, ONBOARDING_BLOCK, cognition_readme) — drift-in-waiting; and the v0.13.0 "~53% cut" accounting excludes the ~200-300-token always-on SERVER_INSTRUCTIONS paid every session and every compact.

Verified coherent: in-process delete cascade end-to-end; update re-embed via shared path; mark_curated no-reembed; connect-vs-compact instructions single constant; journal append atomicity; git-hygiene locking; venv-sync guard; cross-project read attach; hatch force-include already removed.

THEMES: (1) The supported-but-unused separate-clones topology is the least defended (merge replay, dedup, edgeless sweep) — invest in merge-shaped tests + dedup contracts or de-scope it honestly. (2) Repair paths drift from the canonical paths they repair — reconcilers must CALL the writer, never reimplement it. (3) Cross-process discipline inconsistent across structurally identical problems (git_hygiene lock vs Chroma recreate). (4) The context pipeline restores rules but not state on compact. (5) The host contract is held by convention, not validation (empty env var, .env cwd, version sync).
