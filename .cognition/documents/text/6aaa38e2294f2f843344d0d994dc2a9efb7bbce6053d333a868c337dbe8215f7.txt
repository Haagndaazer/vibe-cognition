# Fable Audit — Stage 1: Per-System Audit — 2026-07-01

Confirmed intended purpose: Vibe Cognition is a fully local MCP server plugin for Claude Code that gives a codebase persistent, structured memory — a git-committed knowledge graph (.cognition/journal.jsonl) of decisions, failures, discoveries, constraints, incidents, patterns, workflows, and open tasks. Multi-agent / multi-human team collaboration is first-class. Local embeddings power semantic search; dashboard is secondary. Curation is agent-driven via /vibe-curate by current design. Success: install from marketplace, restart, and within a session or two see Claude recording history, retrieving context at session start, and after /vibe-curate having a browsable linked graph.

Scope: seven parallel Sonnet auditors, one per system: core graph & persistence; documents/tasks/workflows; embeddings & search; MCP server & tools; session lifecycle & install/upgrade; skills layer; dashboard (weighted secondary). 39 raw findings synthesized.

THEMES: (1) The system neither defends nor announces its own memory loss — all defenses are procedural, not coded. (2) Search failures are indistinguishable from "no history" — need an "honest empty" principle. (3) The docstring/skill layer IS the API and has drifted exactly where newcomers/weak models touch first — re-run the recurring tool-surface self-sufficiency audit repo-wide. (4) Coarse closure: multi-item tasks closed by one resolved_by edge silently drop sub-items. (5) Multi-agent is first-class in intent but second-class in verification (multi-process paths get reasoning, single-process paths get tests).
